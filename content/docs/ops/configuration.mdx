title: Configuration
description: Understand .env.local fallback, runtime Supabase connections, bootstrap payloads, and how to reconfigure Ops later.
---

Ops supports two related configuration models: build-time environment fallback for development and runtime Supabase connection storage for desktop app installs.

Source checkouts use `.env.local`. Packaged desktop installs use runtime configuration entered through the app or prepared manually.

## .env.local fallback for development

In a source checkout, Ops resolves its Supabase connection from `.env.local` using the development fallback variables. This keeps local development simple and compatible with existing repo-based workflows.

```dotenv
VITE_SUPABASE_URL=...
VITE_SUPABASE_PUBLISHABLE_DEFAULT_KEY=...
```

The setup wizard (`pnpm ops setup`) creates this file automatically.

## Runtime Supabase connection for installed apps

Installed desktop apps should not depend on source-repository env files. Instead, Ops stores a runtime Supabase connection after first launch when the user saves a connection in the app settings form.

This makes packaged deployments practical because the app can be repointed to another backend without rebuilding the desktop bundle.

## How bootstrap payloads work

<Steps>

### Prepare connection details

The bootstrap payload contains the Supabase URL, publishable key, and optional project reference. This can be prepared manually or by the install script.

### Ops consumes it on first launch

The desktop app reads the payload, persists the connection as runtime config, and removes the one-time bootstrap file.

### Later launches use persisted runtime config

The app no longer depends on the bootstrap file after the connection is stored locally.

</Steps>

<Callout type="info" title="Conceptual model">
Think of the bootstrap payload as a handoff document from the operator workflow into the installed app. It is not the long-term source of truth after first launch.
</Callout>

## How to reconfigure a connection later

For source checkouts, update `.env.local` directly or re-run the setup wizard:

```bash
pnpm ops setup
```

End users can adjust the runtime connection from inside the app when the connection form is exposed in app settings.
