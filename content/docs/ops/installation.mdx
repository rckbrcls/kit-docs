title: Installation
description: Install Ops from source or as a packaged macOS app.
---

Ops installation covers two related but different workflows: source checkout setup for people working in the repository, and packaged desktop installation for people who want a ready-to-run macOS app.

## Source installation

### Recommended source flow

```bash
pnpm install
pnpm ops setup
pnpm ops dev
```

The setup wizard handles prerequisite validation, `.env.local` creation, dependency installation, Supabase project linking, and migration pushing.

## macOS app installation

### Install script

```bash
curl -fsSL https://raw.githubusercontent.com/polterware/ops/main/install.sh | bash
```

The script detects your OS and architecture, downloads the correct release artifact from GitHub, and installs it to `/Applications/`.

For a pinned build:

```bash
curl -fsSL https://raw.githubusercontent.com/polterware/ops/main/install.sh | bash -s -- --version 1.0.0
```

For private repositories, set `GITHUB_TOKEN` before running the script.

## Supabase prerequisites

- A Supabase project that will host authentication, database schema, row-level security, and RPC functions.
- Credentials and access needed for linking and migration workflows.
- A project URL and publishable key that can be used by Ops at runtime.

## Runtime configuration versus .env.local

| | Use .env.local fallback | Use runtime configuration |
|---|---|---|
| **Best fit** | Source checkout development. | Installed desktop apps. |
| **Storage model** | Build-time environment values available during development. | Persisted runtime connection after first-launch configuration or manual setup. |
| **Recommended owner** | Developers and operators working from source. | Operators distributing packaged apps to end users. |
