There are two supported ways to get started with Ops. The right path depends on whether you want to modify the app from source or install a packaged macOS desktop app.

Both flows depend on a Supabase connection. The difference is where the connection comes from: source checkouts use `.env.local`, while installed apps use runtime configuration.

## Quick start for a source checkout

<Callout type="info" title="Recommended path">
Use this path if you are a developer or operator working directly from the repository.
</Callout>

```bash
pnpm install
pnpm ops setup
pnpm ops dev
```

The setup wizard validates prerequisites, prepares `.env.local`, installs dependencies, links the Supabase project, and pushes migrations.

After setup finishes, `pnpm ops dev` is the only command you normally need for local development.

## Quick start for the installed macOS app

<Callout type="info" title="Recommended path">
Use this path if you want to deploy or test the packaged app without rebuilding Ops locally.
</Callout>

```bash
curl -fsSL https://raw.githubusercontent.com/polterware/ops/main/install.sh | bash
```

The install script downloads the latest release from GitHub, extracts it, and moves it to `/Applications/`. On first launch, the app presents a connection form where you enter your Supabase URL and publishable key.

To pin a specific release:

```bash
curl -fsSL https://raw.githubusercontent.com/polterware/ops/main/install.sh | bash -s -- --version 1.0.0
```

## Requirements and first successful launch

### Base requirements

- Node.js 20+ and pnpm for source-based workflows.
- Supabase CLI for repository setup and migration workflows.
- Rust and Tauri system dependencies only if you are running desktop development from source.
- A reachable Supabase project URL and publishable key.

### Confirm the app is ready

<Steps>

### Connection exists

The app should either read `.env.local` in a source checkout or find a runtime connection provided through the app settings.

### Sign-in screen loads

You should reach the login flow without a missing configuration error.

### Business areas open

After authentication, product catalog, inventory, orders, sales, payments, analytics, and role-based access checks should all resolve against Supabase.

</Steps>
