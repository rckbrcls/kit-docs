The Ops CLI handles both local development startup and Supabase workflow operations.

## CLI commands

| Command | Description |
|---|---|
| `pnpm ops` | Interactive menu |
| `pnpm ops setup` | Full setup wizard (prereqs, env, deps, supabase link, migrations) |
| `pnpm ops dev` | Start local development (web or desktop) |
| `pnpm ops db push` | Push migrations to the linked Supabase project |
| `pnpm ops db lint` | Lint migration files |
| `pnpm ops db reset` | Reset linked remote database (destructive) |
| `pnpm ops db local-reset` | Reset local Docker database (destructive) |
| `pnpm ops check` | Run Prettier + ESLint |

## Development mode

```bash
pnpm ops dev
```

This is the development entry point. Use it after the repository has been set up with `pnpm ops setup`.

## Web versus desktop development mode

- Web-only mode starts the Vite-based web experience for fast UI iteration.
- Desktop mode starts the Tauri shell plus the web app so you can verify native desktop behavior.
- Desktop mode requires the Rust toolchain and Tauri prerequisites.
