The command board is Polter's main interface. It organizes commands from Supabase, GitHub, Vercel, and your package manager into feature tabs, lets you build commands interactively, and pin frequently used workflows.

## Feature Tabs

Commands are grouped into 9 tabs based on what you're doing, not which tool they belong to:

| Tab | What it groups |
|---|---|
| ЁЯЧД **Database** | Supabase `db`, `migration`, `seed`, `inspect` |
| тЪб **Functions** | Supabase edge functions |
| ЁЯЪА **Deploy** | Vercel deploy/promote/rollback, Supabase db/functions |
| ЁЯЧГя╕П **Repo** | GitHub repos, PRs, issues, releases |
| ЁЯУж **Packages** | npm/pnpm/yarn/bun: install, add, remove, build, publish, audit, scripts |
| ЁЯФД **CI/CD** | GitHub workflows/runs, Vercel env vars |
| ЁЯФР **Auth & Storage** | Supabase storage, secrets, SSO |
| ЁЯМР **Networking** | Supabase and Vercel domains, SSL, network restrictions |
| тЪЩя╕П **Setup** | Login and linking for all tools |

Some commands appear in multiple tabs when they're relevant to more than one workflow.

## Interactive Command Builder

When you select a command, Polter walks you through:

1. **Suggested args** тАФ common subcommands shown as selectable options
2. **Custom args** тАФ free-form text input for anything not listed
3. **Flags** тАФ tool-specific flags presented as checkboxes

The final command is assembled and shown for confirmation before execution.

## Flags

Flags vary by tool and are presented in the flag picker after arg selection:

**Supabase:** `--debug`, `--yes`, `--create-ticket`, `--experimental`

**Vercel:** `--yes`, `--debug`

**Package Manager:** `--save-dev`, `--global`, `--dry-run`

## Pinned Commands and Runs

Two pinned sections sit at the top of the board:

- **Pinned Commands** тАФ base commands like `db` or `deploy` for quick access
- **Pinned Runs** тАФ exact commands with args and flags, like `db pull --yes`

Press `тЖТ` on a command to pin/unpin it. After a successful execution, Polter can also pin that exact run.

Pins are stored locally using OS-level app config and persist across sessions.

## Custom Command Mode

Bypass the guided flow and type raw args for any tool:

```
-v
status -o json
db pull
projects list
```

## Shell Execution

Polter resolves each tool binary before running it:

- **supabase** тАФ resolved from the current repository first (`node_modules/.bin`), then `PATH`
- **gh, vercel** тАФ resolved from `PATH`
- **npm/pnpm/yarn/bun** тАФ auto-detected from lock file (`bun.lockb` тЖТ bun, `pnpm-lock.yaml` тЖТ pnpm, `yarn.lock` тЖТ yarn, `package-lock.json` тЖТ npm)

All commands execute as:

```bash
<tool> <command> <extra-args> <flags>
```
