Polter is an infrastructure orchestrator CLI for managing dev processes, pipelines, CLI commands (Supabase, GitHub, Vercel, Git), and packages from one unified interface. Instead of juggling multiple CLIs, you browse a tabbed command board, chain steps into reusable pipelines, manage background processes, and apply declarative project configuration — all from a single TUI.

## Installation

### Run one-off without installing

```bash
npx @polterware/polter@latest
```

### Install globally

```bash
npm install -g @polterware/polter
```

Then run:

```bash
polter
```

Use a global install to have Polter available across every repository.

### Set Up AI Integration

```bash
polter setup          # Install Claude Code skill
polter mcp install    # Register MCP server
```

This installs the `/polter` skill for Claude Code and registers Polter as an MCP server. See [MCP / AI Integration](/docs/polter/mcp) for details.

### Update

```bash
npm install -g @polterware/polter@latest
```

You can also update from inside Polter — go to **Actions > Update Polter**.

## Requirements

- **Node.js** >= 18

CLI tools are optional. Polter detects which ones are installed and shows their status on the [Tool Status](/docs/polter/tool-status) screen:

- **Supabase CLI** — [Install guide](https://supabase.com/docs/guides/cli)
- **GitHub CLI (`gh`)** — [Install guide](https://cli.github.com/)
- **Vercel CLI** — [Install guide](https://vercel.com/docs/cli)
- **Package manager** — auto-detected from your lock file (npm, pnpm, yarn, or bun). No extra configuration required.

Check your environment:

```bash
node -v
supabase --version
gh --version
vercel --version
```

## Typical Flow

1. Choose a command from the board
2. Optionally pin/unpin with `→`
3. Choose suggested args or type custom extra args
4. Pick optional flags
5. Confirm and execute
