## Interactive Command Builder

Guided flow for selecting a command, subcommand, and extra arguments. Instead of typing full Supabase CLI commands from memory, Polterbase walks you through each step interactively.

## Suggested Subcommand Picker

Select common args (for example `db pull`) from boxed sections before typing custom args. This reduces typos and helps discover subcommands you may not know about.

## Unified Command Board

Pinned runs, pinned commands, grouped categories, and actions are displayed in boxed sections on one screen. The board gives you a single view of everything available.

## Global Flags Picker

Add common global flags in one step. Available flags:

| Flag | Description |
|---|---|
| `--create-ticket` | Create a support ticket on error |
| `--debug` | Enable debug logs |
| `--experimental` | Enable experimental features |
| `--yes` | Auto-confirm prompts |

## Pinned Commands and Runs

Polterbase supports two pinned sections at the top of the main menu:

- **Pinned Runs** for exact commands like `db pull --debug`
- **Pinned Commands** for base commands like `db` or `start`

Use `→` on a selected base command to pin or unpin it. Use `→` on the suggested subcommand screen to pin exact runs before executing. After a successful execution, Polterbase can also pin that exact command into Pinned Runs.

Pins are persisted locally using OS-level app config storage.

## Custom Command Mode

Run raw Supabase arguments like `-v` or `status -o json`. Use this mode for free-form args when the guided flow is not needed.

Examples:

```bash
-v
status -o json
db pull
projects list
```

## Built-in Self-Update

Update Polterbase for the current repository or globally through npm, directly from the CLI:

1. Go to the **Actions** section
2. Choose **Update Polterbase**
3. Choose **Current repository** or **Global install**
4. Confirm the npm update command

## Shell Execution

Polterbase resolves the `supabase` binary from the current repository first, then falls back to `PATH`. This ensures repository-local installations take precedence.

All workflow commands are executed as:

```bash
supabase <command> <extra-args> <global-flags>
```
